Step2 Vue-Router路由分析

  1. 2023-03-15
  2. 482字
  3. 2 min read

vue-router
前端所谓的路由：kv键值对
key：URL（地址栏中的路径）
value：相应的路由组件

如何拆分路由？

路由组件：

  1. 页面切换始终不变的可以定义为非路由组件，例如头部和底部
  2. 变换的部分设计为非路由组件，key、value值发生变化
  3. 分析出每个页面有那些路由

有些页面有某组件，有些页面没有某组件，写代码的时候考虑v-if，v-show实现

项目实现步骤

  1. 书写静态页面（HTML+CSS）
  2. 拆分路由
  3. 获取服务器的数据动态展示
  4. 完成动态业务逻辑

路由组件的搭建

vue-router

  • components文件夹：经常放置路由组件
  • pages|views文件夹：经常放置非路由组件

配置路由

项目中的路由信息一般放在routes文件夹中
src/routes/index.js

  2. 使用路由组件
  3. 配置路由信息

声明式和编程式

路由元信息的使用

 routers: [{
            // 一级路由
            path: "/home",
            component: Home,
            //路由元信息
            meta: { show: true }
        }}]

 // 纯字符串形式传参
        // params参数
        // this.$router.push("/search"+this.keyword);
        // query参数
        // this.$router.push("/search"+this.keyword+"?k="+this.keyword.toUpperCase());
        // 模板字符串
        // this.$router.push(`/search/${this.keyword}?k=${this.keyword.toUpperCase()}`)
        // 对象传参
        this.$router.push({name:"search",params:{keyword:this.keyword},query:{k:this.keyword.toUpperCase()}})

路由传递参数（对象写法）path是否可以结合params参数一起使用？

路由跳转传参的时候，对象的写法可以是name，path形式，但是需要注意的是，path的这种写法不能和params参数一起使用
//path和params一起使用的情况
this.$router.push({path:''/search},params:{keyword:this.keyword},query:{k:this.keyword.toUpperCase()})
